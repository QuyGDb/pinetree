01 — Merge

Git Merge

Gộp nhánh feature vào main bằng cách tạo thêm một merge commit — giữ nguyên toàn bộ lịch sử của cả 2 nhánh.

main và feature đang phát triển song song — chưa được gộp
main: C1 → C2 → C3 (vẫn đang tiến)
feature: tách từ C1, có thêm F1 → F2
Hai nhánh hoàn toàn độc lập với nhau
Sau git merge feature — một merge commit M được tạo ra, nối 2 nhánh lại
M = Merge Commit: có 2 parent (C3 và F2)
Lịch sử main vẫn nguyên: C1→C2→C3
Lịch sử feature vẫn nguyên: C1→F1→F2
🔀
2 nhánh tồn tại độc lập
Mỗi dev có thể làm việc riêng mà không ảnh hưởng nhau. Nhưng muốn kết hợp thì cần git merge.
✅
Merge commit M có 2 "cha"
Đây là điểm đặc biệt của merge commit — nó ghi nhớ lịch sử từ cả 2 nhánh. Dùng git log --graph để thấy hình dạng này.

Git Rebase

03 — Reset

Git Reset

Di chuyển con trỏ HEAD về một commit cũ hơn. Điều gì xảy ra với code của bạn phụ thuộc vào 3 chế độ dưới đây.

📌 Kịch bản: bạn có 4 commits, muốn reset về C2
main
C1
C2 ← target
C3
C4 HEAD
Lệnh thực thi: git reset [--soft | --mixed | --hard] C2  ·  C3 và C4 sẽ biến mất khỏi git log — nhưng nội dung thay đổi sẽ đi đâu?
Git Log sau reset
a1b2c3 C1 — initial commit
d4e5f6 C2 — stable feature HEAD
7g8h9i C3 — add login page
j1k2l3 C4 — fix login bug
Nội dung thay đổi của C3 & C4 đi đâu?
🗄️ Repository (git log)
✓ C1, C2 còn trong lịch sử
✗ C3, C4 bị xóa khỏi lịch sử
📋 Staging Area (git status)
— trống —
📁 Working Directory (ổ cứng của bạn)
— đã xóa sạch! —

04 — Revert

Git Revert

Không xóa lịch sử. Tạo thêm một commit mới có nội dung ngược lại với commit lỗi — an toàn cho teamwork.

main
C1
C2 ⚠ lỗi
C3
R revert C2
HEAD HEAD
❌ Reset — nguy hiểm khi đã push

Xóa commit khỏi lịch sử. Người khác đang pull sẽ bị conflict thảm họa.

✅ Revert — an toàn cho team

Thêm commit mới, lịch sử không bị thay đổi. Mọi người pull bình thường.

05 — So Sánh

Khi nào dùng lệnh nào?

Chọn đúng công cụ cho đúng tình huống.

Tình huống Lệnh Lý do
Lỡ commit sai, chưa push reset --soft Sửa lại code & commit lại gọn gàng.
Code lỗi đã push lên server chung revert Mọi người thấy lịch sử đảo ngược, không gây conflict.
Muốn xóa sạch thay đổi nháp reset --hard Dọn dẹp nhanh — cẩn thận, không hoàn tác được!
Gộp code sạch trước khi tạo PR rebase Lịch sử tuyến tính, dễ review hơn.
Giữ toàn bộ lịch sử khi merge nhánh merge Tường minh, rõ khi nào nhánh được gộp.
